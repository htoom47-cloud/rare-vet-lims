# Bug Tracker — Rare Vet LIMS UAT

**Project:** Rare Vet LIMS v1.0  
**Laboratory:** AL NAWADER VETERINARY CARE CENTER  
**Purpose:** Register and track defects found during UAT and post-go-live

---

## How to use

1. Assign the next sequential **Bug ID** (`BUG-001`, `BUG-002`, …).
2. Fill all columns when logging a defect.
3. Link to **UAT Test ID** in Notes when applicable (e.g. `UAT-CUS-003`).
4. Update **Status** as work progresses: `Open` → `In Progress` → `Fixed` → `Verified` → `Closed`.
5. Do not delete rows — mark `Closed` or `Won't Fix` with justification.

---

## Severity definitions

| Severity | Definition |
|----------|------------|
| **Critical** | System down, data loss, security breach, cannot register samples or release reports |
| **High** | Major feature broken, no workaround |
| **Medium** | Feature impaired, workaround exists |
| **Low** | Cosmetic, typo, minor UX |

## Priority definitions

| Priority | Definition |
|----------|------------|
| **P0** | Fix before go-live |
| **P1** | Fix in first week post go-live |
| **P2** | Fix in v1.1 |
| **P3** | Backlog |
