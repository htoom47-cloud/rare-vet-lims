# Reference Quality Report

> Generated by `GET /api/reference-ranges/quality-audit` or `GET /api/report-mastering/quality-audit` (when `REPORT_MASTERING_ENABLED=true`).
> Run migration first: `node backend/src/scripts/migrate.js`

## Purpose

Admin audit for reference ranges, parameter translations, and device mapping consistency before publishing lab reports.

## Checks performed

| Check | Description |
|-------|-------------|
| **Missing range** | Active report parameters without a reference range for each species |
| **Inverted range** | `min_value > max_value` |
| **Unit mismatch** | Range unit differs from parameter unit |
| **Flag without range** | Validated results flagged HIGH/LOW/CRIT with no active reference |
| **Duplicate ranges** | Same parameter + species + device + sex |
| **Missing Arabic** | Parameters shown in report without `name_ar` |
| **Missing device code** | No `device_parameter_mappings` entry and no `device_code` / `short_code` |

## API

```http
GET /api/reference-ranges/quality-audit?species=camel&test_id=<uuid>
GET /api/report-mastering/quality-audit?species=camel   # requires REPORT_MASTERING_ENABLED=true
```

Response shape:

```json
{
  "success": true,
  "data": {
    "generatedAt": "2026-07-03T...",
    "summary": {
      "missingRange": 0,
      "invertedRanges": 0,
      "unitMismatches": 0,
      "flagWithoutRange": 0,
      "duplicateRanges": 0,
      "missingArabic": 0,
      "missingDeviceCode": 0
    },
    "missingRange": [],
    "invertedRanges": [],
    "unitMismatches": [],
    "flagWithoutRange": [],
    "duplicateRanges": [],
    "missingArabic": [],
    "missingDeviceCode": []
  }
}
```

## Admin UI

- **Reference Ranges** (`/reference-ranges`) — CRUD for species/device/test/parameter ranges; client validates Min ≤ Max.
- **Report Mastering** (`/report-mastering`) — Parameters, device mappings, quality audit tab (feature flag).

## Report behaviour (always on)

- Reference column shows **N/A** / **غير متوفر** instead of `-` when no range exists.
- **HIGH/LOW** flags are suppressed when no reference range is configured.
- Parameter codes on the report come from **device mapping** → `device_code` → `short_code` → system code (fallback).
